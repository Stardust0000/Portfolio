Case Study — Daily Quest

  • Role: Frontend Developer
  • Type: React Web Application
  • Tech: React • JavaScript • LocalStorage • CSS
  • Status: Code Complete

Problem

Most beginner task apps focus on CRUD functionality but fail to provide visual motivation or progress awareness, resulting in poor user retention.

Goal

Build a clean, responsive task-tracking application that:

  • feels lightweight and engaging
  • persists data without a backend
  • uses React state effectively

Solution

I built Daily Quest, a React-based task manager designed around progress visibility and simplicity.

  • Component-driven UI using React hooks
  • LocalStorage persistence for offline usage
  • Dynamic progress tracking and state updates

Outcome

  • Strengthened understanding of React state and lifecycle
  • Improved component reusability and UI logic separation
  • Production-ready structure for future scaling

Case Study — Twilight Trails

  • Role: Frontend Developer
  • Type: Responsive Web Experience
  • Tech: HTML • CSS • JavaScript
  • Status: Live

Problem

Most travel websites overload users with cluttered layouts and generic visuals, reducing engagement and discoverability—especially on smaller screens.

Goal

  • Create an atmospheric, premium dark-themed experience
  • Ensure seamless responsiveness across devices
  • Use minimal JavaScript for smooth interactions

Solution

I built Twilight Trails, a multi-page responsive website focused on visual storytelling and clarity.

  • Dark UI with high-contrast typography for readability
  • Custom JavaScript slideshows instead of third-party libraries
  • Modular layout for scannable, structured content

Key Features

  • Responsive landing and main pages
  • Custom slideshow & carousel logic (vanilla JS)
  • Clean navigation with consistent visual hierarchy
  • GitHub Pages deployment

Outcome

  • Translated UI concepts into functional layouts
  • Wrote clean, reusable JavaScript
  • Delivered a production-ready static website

Case Study — ToonFlix

  • Role: UI/UX Designer
  • Type: Mobile UX Concept
  • Tools: Figma • Wireframing • UX Research
  • Status: Concept Design

Problem

Many entertainment apps overwhelm first-time users with cluttered onboarding and unclear content discovery paths.

Goal

Design a mobile-first entertainment app concept that:

  • simplifies onboarding
  • prioritizes content discovery
  • maintains visual hierarchy on small screens

Solution

I designed ToonFlix, a mobile UX concept focused on clarity and flow rather than visual excess.

  • Low-friction onboarding screens
  • Clear navigation patterns
  • Consistent spacing and typography

Outcome

  • Improved understanding of mobile UX constraints
  • Hands-on experience with wireframing and flow design
  • Foundation for future high-fidelity iterations
